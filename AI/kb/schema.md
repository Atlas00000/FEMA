# FEMA knowledge base schema

## Runs (INF-RUN)

Canonical layout and `run_id` format: [`runs/README.md`](runs/README.md).

`metrics.json` required fields:

| Field | Type | Notes |
| ----- | ---- | ----- |
| `run_id` | string | Folder name |
| `registered_at` | ISO-8601 UTC | Wall clock |
| `preset` | string | e.g. `FEMA_EURUSD_M5_PRODUCTION` |
| `preset_slug` | string | Short token used in `run_id` |
| `symbol` | string | |
| `timeframe` | string | |
| `window.start` / `window.end` | `YYYY.MM.DD` | |
| `window_hash` | string | 8 hex |
| `role` | enum | `lock` · `demo` · `collect` · `candidate` · `reject` · `other` |
| `baskets_path` | string | Repo-relative preferred |
| `ea_run_id` | string\|null | From EA meta if present |
| `metrics` | object | At least `n`, `profit_factor`, `win_rate`, `net` |
| `status` | string | `recorded` · `superseded` · `archived` |
| `notes` | string | Free text |

## Candidates (INF-PRESET / EL2)

File: [`candidates.csv`](candidates.csv) — regenerated by `python -m fema_ops gate-polish` from Discovery seed + run_id map.

| Column | Notes |
| ------ | ----- |
| `id` | Preset / candidate id |
| `parent` | Usually `PRODUCTION` |
| `subsystem` | Search-map id (`session`, `adx`, …) |
| `diffs` | Human/machine diff summary |
| `window` / `symbol` | Test scope |
| `pf` / `wr` / `dd` | Filled after tester (`wr` as fraction 0–1) |
| `status` | `lock` · `candidate` · `reject` · `control` · `queued` (+ legacy `tested`/`pass`/`fail`) |
| `failure_reason` | Enum: `session` · `regime` · `dd_breach` · `pf_breach` · `other` (+ legacy labels) |
| `run_id` | INF-RUN registry id when tested |
